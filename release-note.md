## 0.8.0

### Minor Changes

- **稳定的按钮标识（actionKey）**：按钮 ID 体系重构。新增 `actionKey` 字段，基于按钮内容（标题+类型+功能值）的哈希值，跨预设稳定。钩子的 `match.key` 改为匹配 `actionKey`，预设切换后钩子自动跟随，不再因 ID 随机失效。
- **钩子匹配条件智能锁定**：选择「匹配 - 按钮」后，其余条件（类型、标题包含、功能值）自动灰显并清空，避免误设冲突的 AND 逻辑。
- **MCP 文档修正**：`set-click-hook` 的 `match.key` 说明从"Button ID"修正为"Button actionKey"，`trigger()` 参数增加 `id` 与 `actionKey` 的区分提示。

### Patch Changes

- **修复 kernel 启动报错 `Invalid module`**：将 ID/actionKey 生成函数抽取为 `src/hash.ts`（零依赖），`kernel.ts` 改为从 `hash.ts` 导入，避免内联浏览器端代码导致 goja 运行时报错。
- **消除哈希实现重复**：之前 `utils.ts` 和 `kernel.ts` 各维护一份相同的 hash 函数，存在静默不同步的风险。统一收敛到 `src/hash.ts`。

### Minor Changes

- **全局点击钩子系统**：现在可以为导航按钮的点击行为挂载「前置钩子」、「后置钩子」和「替换钩子」。你可以编写自定义脚本在点击前拦截（如确认弹窗）、点击后响应（如记录日志），甚至完全替换按钮的原始行为（如放烟花庆祝）。钩子支持按按钮 ID、类型、标题匹配，可独立开关和设置优先级。
- **二级菜单定制外观**：导航栏的二级菜单弹窗现在支持独立的样式自定义（背景色、圆角、悬停高亮、阴影、图标大小等），与导航栏主样式分离，在设置面板的「外观」→「二级菜单样式」中调整。
- **二级菜单 MCP 完整支持**：AI 助手可以通过 MCP 工具创建带二级菜单的分组按钮（`add-action` 支持 `children` 和 `submenuLayout`），并统一设置其外观样式。
- **编辑器命令智能校验**：MCP 创建编辑器命令按钮时，自动检测用户是否已配置对应快捷键，只暴露已设快捷键的命令，避免创建不可用的按钮。前端也增加兜底搜索和区分度更高的错误提示。

### Patch Changes

- **新增「滚动到底部」**：与「返回顶部」对称的新增内置命令，一键滚动到当前文档末尾。
- **修复「返回顶部」不生效**：修正了可滚动容器的继承问题，`scrollTo` 直接作用于正确的 DOM 元素。
- **修复二级菜单样式不生效**：`--submenu-*` CSS 变量因 DOM 层级关系未传递到 Submenu 组件，改为通过 `inlineStyle` prop 直接注入。
- **修复设置页打开报错**：`ButtonSettings` 组件访问了 `plugin` 但未导入，导致 `ReferenceError`。
- **修复内置命令执行出错**：同步返回的 `execute` 函数因 `.catch()` 调用在 `undefined` 上抛错，用 `Promise.resolve()` 包裹修复。
